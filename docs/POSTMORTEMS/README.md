# Postmortems

Purpose: capture learnings from each game sprint and feed them back into the template.

## When to write one
- After shipping an MVP
- After a major refactor
- After a production incident or repeated bug pattern

## Template
### Summary
- What was built
- Timebox (dates)
- Outcome

### What worked
- Bullet list

### What broke
- Bullet list

### Root causes
- Bullet list

### Template improvements
- Specific PR-sized items to add/change in template

### Follow-ups
- Owners and next steps
